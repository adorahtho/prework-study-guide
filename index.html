HMTL

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Child elements are nested under the parent element.
  • HTML elements contain an opening and closing tag.
  • There are 6 total levels of headings ranging from large, "h1", to small, "h6", that should be used in order from h1 to h6. "h1" should only appear once on each page.
  • There are 2 different types of lists (li), unordered lists and ordered lists. The item listed is nested under the parents element/type of list. (ul and ol)
  • Image elements contain various attributes, but the main one is source. (src)
  • The line break element is used to create empty space between elements. (br)
  • The anchor element is used for creating links to the same or different webpage. (a)
  • The section element is used to organize and seperate different parts of the page.
  • The footer element contains information, such as, information about the creator of the site. (footer)

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS applies to a specific HTML tag.
  • Internal Style Sheet can be added to the head element with the (style) element and only applies to that webpage. One change will apply to all identified (id) elements with the same "class" attribute.
  • The most common way to style is using external CSS Style Sheet to keep all CSS rules seperate and easily change styles if needed. The external CSS Style Sheet can be linked inside the HTML head tag to apply changes to the webpage.
  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: indicates the changes we want to add
  • git commit -m "add message here": a message describing the changes made
  • git pull origin (branch name): pulling the branch's modifications into the local environment
  • git push origin (branch name): pushing the changes we have locally to our remote GitHub branch

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • String includes topics surrounded by quotation marks.
  • console.log(argument): method used to output a message to the web console by adding an argument.
  • Chrome DevTools is used to demonstrate how console.log works and to confirm that the script tag added to index.html properly connects with script.js
  • Boolean is a data type with a truthy or falsy value.
  • Array is a single variable used to hold a group of data that's related in some way.
  • Index is a unique number used to identify the location of the data item in the array.
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
